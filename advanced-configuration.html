تصميم الشبكات المتقدم

تصميم الشبكة هو الخطوة الأولى والأهم في بناء بنية تحتية قوية وموثوقة. يتطلب التصميم الجيد فهمًا عميقًا لمتطلبات المؤسسة واحتياجاتها المستقبلية.

مبادئ تصميم الشبكات:

  • الهرمية (Hierarchy): تقسيم الشبكة إلى طبقات (الوصول، التوزيع، النواة) لتسهيل الإدارة والتوسع.
  • التكرار (Redundancy): توفير مسارات بديلة لضمان استمرارية الخدمة في حالة فشل أحد المكونات.
  • القابلية للتوسع (Scalability): تصميم الشبكة بحيث يمكن توسيعها بسهولة لاستيعاب النمو المستقبلي.
  • الأمان (Security): دمج الأمان في كل مستوى من مستويات الشبكة.
  • قابلية الإدارة (Manageability): تسهيل مراقبة الشبكة وإدارتها وتحديثها.

نموذج التصميم الهرمي ثلاثي الطبقات:

1. طبقة النواة (Core Layer)

العمود الفقري للشبكة، مسؤولة عن نقل كميات كبيرة من البيانات بسرعة وكفاءة.

  • تتكون من أجهزة توجيه وتبديل عالية الأداء
  • تركز على السرعة والموثوقية
  • لا تقوم بمعالجة حزم البيانات أو تصفيتها
  • توفر التكرار لضمان توافر الشبكة

2. طبقة التوزيع (Distribution Layer)

تعمل كوسيط بين طبقتي النواة والوصول، وتوفر وظائف التوجيه والتصفية والسياسات.

  • تنفذ سياسات الشبكة (التصفية، QoS، الأمان)
  • تقوم بتجميع اتصالات طبقة الوصول
  • توفر التوجيه بين الشبكات الفرعية المختلفة
  • تحد من نطاق الفشل في الشبكة

3. طبقة الوصول (Access Layer)

نقطة اتصال المستخدمين والأجهزة بالشبكة.

  • توفر اتصال للمستخدمين النهائيين والأجهزة
  • تنفذ أمان المنفذ وVLAN
  • تدعم PoE (الطاقة عبر الإيثرنت) للأجهزة مثل الهواتف ونقاط الوصول اللاسلكية
  • تنفذ ميزات مثل الحماية من العواصف البث

الشبكات الافتراضية المحلية (VLANs)

الشبكات الافتراضية المحلية (VLANs) هي تقنية تسمح بتقسيم شبكة محلية فعلية واحدة إلى عدة شبكات منطقية منفصلة. تعتبر VLANs أساسية في تصميم الشبكات الحديثة.

فوائد استخدام VLANs:

  • تحسين الأداء: تقليل نطاق البث وتقسيم مجالات التصادم.
  • تعزيز الأمان: عزل حركة المرور بين المجموعات المختلفة.
  • تبسيط الإدارة: تنظيم الشبكة منطقيًا بدلاً من فعليًا.
  • تقليل التكاليف: الاستفادة القصوى من البنية التحتية الحالية.
  • المرونة: إمكانية نقل المستخدمين منطقيًا دون تغيير الاتصالات الفعلية.

أنواع VLANs:

1. VLAN قائم على المنفذ (Port-based VLAN)

يتم تعيين كل منفذ في المحول إلى VLAN محدد. هذا هو النوع الأكثر شيوعًا.

# مثال على تكوين VLAN قائم على المنفذ في Cisco IOS
Switch# configure terminal
Switch(config)# vlan 10
Switch(config-vlan)# name Engineering
Switch(config-vlan)# exit
Switch(config)# vlan 20
Switch(config-vlan)# name Marketing
Switch(config-vlan)# exit

# تعيين المنافذ إلى VLANs
Switch(config)# interface range fastethernet 0/1-5
Switch(config-if-range)# switchport mode access
Switch(config-if-range)# switchport access vlan 10
Switch(config-if-range)# exit
Switch(config)# interface range fastethernet 0/6-10
Switch(config-if-range)# switchport mode access
Switch(config-if-range)# switchport access vlan 20

2. VLAN قائم على العنوان MAC (MAC-based VLAN)

يتم تعيين الأجهزة إلى VLANs بناءً على عناوين MAC الخاصة بها، بغض النظر عن المنفذ المتصلة به.

3. VLAN قائم على البروتوكول (Protocol-based VLAN)

يتم تعيين الحزم إلى VLANs بناءً على نوع البروتوكول (مثل IPv4، IPv6، AppleTalk).

ربط VLANs (VLAN Trunking)

لنقل حركة مرور من عدة VLANs عبر رابط واحد بين المحولات، نستخدم تقنية الربط (Trunking).

# مثال على تكوين منفذ ربط في Cisco IOS
Switch(config)# interface gigabitethernet 0/1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 10,20,30
Switch(config-if)# switchport trunk native vlan 1

بروتوكولات الربط الشائعة:

  • IEEE 802.1Q: معيار مفتوح يضيف علامة إلى إطارات الإيثرنت لتحديد VLAN.
  • ISL (Inter-Switch Link): بروتوكول خاص بشركة Cisco (قديم ولم يعد يستخدم كثيرًا).

التوجيه بين VLANs (Inter-VLAN Routing)

للسماح بالاتصال بين VLANs المختلفة، نحتاج إلى جهاز توجيه أو محول طبقة 3.

طرق التوجيه بين VLANs:

1. التوجيه التقليدي (Router-on-a-stick)

استخدام جهاز توجيه خارجي مع واجهة فرعية لكل VLAN.

# مثال على تكوين Router-on-a-stick في Cisco IOS
Router(config)# interface fastethernet 0/0
Router(config-if)# no shutdown
Router(config-if)# exit

Router(config)# interface fastethernet 0/0.10
Router(config-subif)# encapsulation dot1q 10
Router(config-subif)# ip address 192.168.10.1 255.255.255.0
Router(config-subif)# exit

Router(config)# interface fastethernet 0/0.20
Router(config-subif)# encapsulation dot1q 20
Router(config-subif)# ip address 192.168.20.1 255.255.255.0
2. التوجيه باستخدام محول طبقة 3 (Layer 3 Switch)

استخدام محول متقدم يمكنه أداء وظائف التوجيه.

# مثال على تكوين التوجيه بين VLANs في محول طبقة 3
Switch(config)# ip routing

# تكوين واجهات SVI (Switched Virtual Interface)
Switch(config)# interface vlan 10
Switch(config-if)# ip address 192.168.10.1 255.255.255.0
Switch(config-if)# no shutdown
Switch(config-if)# exit

Switch(config)# interface vlan 20
Switch(config-if)# ip address 192.168.20.1 255.255.255.0
Switch(config-if)# no shutdown

بروتوكولات التوجيه المتقدمة

بروتوكولات التوجيه هي الآليات التي تستخدمها أجهزة التوجيه لتبادل معلومات التوجيه وتحديد أفضل المسارات لتوجيه حركة المرور.

1. OSPF (Open Shortest Path First)

بروتوكول توجيه حالة الرابط يستخدم خوارزمية Dijkstra لحساب أقصر مسار أولاً.

المميزات:

  • يتكيف بسرعة مع التغييرات في الشبكة
  • يدعم VLSM (أقنعة الشبكة الفرعية ذات الطول المتغير)
  • يستخدم مفهوم المناطق لتحسين قابلية التوسع
  • يدعم المصادقة لتأمين تبادل معلومات التوجيه

تكوين OSPF الأساسي:

# مثال على تكوين OSPF في Cisco IOS
Router(config)# router ospf 1
Router(config-router)# network 192.168.1.0 0.0.0.255 area 0
Router(config-router)# network 192.168.2.0 0.0.0.255 area 0
Router(config-router)# default-information originate

مفهوم المناطق في OSPF:

يستخدم OSPF مفهوم المناطق لتقسيم الشبكة إلى أجزاء أصغر لتحسين الأداء وتقليل حجم جداول التوجيه.

  • المنطقة الصفرية (Area 0): منطقة العمود الفقري التي تتصل بها جميع المناطق الأخرى.
  • المناطق العادية (Regular Areas): تتلقى معلومات كاملة عن الشبكة.
  • المناطق الطرفية (Stub Areas): لا تتلقى إعلانات LSA خارجية، مما يقلل حجم قاعدة بيانات حالة الرابط.
  • المناطق الطرفية التامة (Totally Stubby Areas): تتلقى فقط مسار افتراضي إلى المنطقة الصفرية.

2. BGP (Border Gateway Protocol)

بروتوكول توجيه المسار الخارجي الأساسي المستخدم في الإنترنت، يستخدم لتبادل معلومات التوجيه بين الأنظمة المستقلة (AS).

المميزات:

  • يدعم سياسات التوجيه المعقدة
  • يستخدم سمات المسار لاتخاذ قرارات التوجيه
  • يوفر استقرارًا وقابلية للتوسع للشبكات الكبيرة
  • يدعم التجميع والتصفية لتقليل حجم جداول التوجيه

تكوين BGP الأساسي:

# مثال على تكوين BGP في Cisco IOS
Router(config)# router bgp 65000
Router(config-router)# neighbor 192.168.1.2 remote-as 65001
Router(config-router)# network 192.168.100.0 mask 255.255.255.0
Router(config-router)# network 192.168.200.0 mask 255.255.255.0

أنواع جلسات BGP:

  • iBGP (internal BGP): جلسات BGP بين أجهزة توجيه في نفس النظام المستقل.
  • eBGP (external BGP): جلسات BGP بين أجهزة توجيه في أنظمة مستقلة مختلفة.

3. EIGRP (Enhanced Interior Gateway Routing Protocol)

بروتوكول توجيه متقدم خاص بشركة Cisco، يجمع بين ميزات بروتوكولات حالة الرابط وبروتوكولات متجه المسافة.

المميزات:

  • تقارب سريع
  • استخدام فعال لعرض النطاق الترددي
  • دعم VLSM والتوجيه غير المتجانس
  • استخدام خوارزمية DUAL (Diffusing Update Algorithm) لحساب المسارات

تكوين EIGRP الأساسي:

# مثال على تكوين EIGRP في Cisco IOS
Router(config)# router eigrp 100
Router(config-router)# network 192.168.1.0 0.0.0.255
Router(config-router)# network 192.168.2.0 0.0.0.255
Router(config-router)# no auto-summary

تقنيات الشبكات الواسعة (WAN)

الشبكات الواسعة (WAN) تربط المواقع المتباعدة جغرافيًا. هناك العديد من التقنيات المستخدمة لإنشاء اتصالات WAN.

1. MPLS (Multiprotocol Label Switching)

تقنية توجيه عالية الأداء تستخدم تسميات لتوجيه حزم البيانات عبر الشبكة.

المميزات:

  • أداء عالٍ وتأخير منخفض
  • دعم لجودة الخدمة (QoS)
  • قابلية للتوسع وسهولة الإدارة
  • دعم للشبكات الافتراضية الخاصة (VPN)

كيفية عمل MPLS:

  1. يتم تعيين تسمية (label) للحزم عند دخولها شبكة MPLS.
  2. يتم توجيه الحزم بناءً على التسمية بدلاً من عنوان IP الوجهة.
  3. يتم إزالة التسمية عند خروج الحزم من شبكة MPLS.

2. SD-WAN (Software-Defined WAN)

تطبيق لمفاهيم الشبكات المعرفة بالبرمجيات (SDN) على شبكات WAN، يسمح بإدارة مركزية وذكية للشبكة.

المميزات:

  • تقليل تكاليف WAN من خلال استخدام اتصالات الإنترنت العامة
  • تحسين الأداء من خلال التوجيه الذكي
  • تبسيط الإدارة من خلال واجهة مركزية
  • نشر سريع للمواقع الجديدة

مكونات SD-WAN:

  • وحدة تحكم SD-WAN: الدماغ المركزي للشبكة، يوفر إدارة وتحكم مركزيين.
  • أجهزة SD-WAN الطرفية: أجهزة توجيه ذكية في المواقع المختلفة.
  • تطبيقات وخدمات: توفر وظائف مثل التشفير، جدران الحماية، وتحسين WAN.

3. VPN (Virtual Private Network)

تقنية تسمح بإنشاء اتصال آمن عبر شبكة عامة مثل الإنترنت.

أنواع VPN:

Site-to-Site VPN

يربط مواقع كاملة معًا، مثل ربط فروع الشركة بالمقر الرئيسي.

بروتوكولات شائعة:
  • IPsec: مجموعة من البروتوكولات لتأمين اتصالات IP.
  • GRE over IPsec: يجمع بين تغليف GRE وتشفير IPsec.
  • DMVPN (Dynamic Multipoint VPN): يسمح بإنشاء اتصالات مباشرة بين المواقع.
# مثال على تكوين IPsec VPN في Cisco IOS
! تعريف مجموعة التحويل (Transform Set)
Router(config)# crypto ipsec transform-set TS esp-aes 256 esp-sha-hmac
Router(config)# crypto ipsec security-association lifetime seconds 3600

! تعريف سياسة ISAKMP
Router(config)# crypto isakmp policy 10
Router(config-isakmp)# encryption aes 256
Router(config-isakmp)# hash sha
Router(config-isakmp)# authentication pre-share
Router(config-isakmp)# group 5
Router(config-isakmp)# lifetime 86400
Router(config-isakmp)# exit

! تعريف مفتاح مشترك مسبقًا
Router(config)# crypto isakmp key StrongPassword address 203.0.113.1

! تعريف قائمة وصول لتحديد حركة المرور المراد تشفيرها
Router(config)# access-list 101 permit ip 192.168.1.0 0.0.0.255 192.168.2.0 0.0.0.255

! تعريف خريطة التشفير
Router(config)# crypto map CMAP 10 ipsec-isakmp
Router(config-crypto-map)# set peer 203.0.113.1
Router(config-crypto-map)# set transform-set TS
Router(config-crypto-map)# match address 101
Router(config-crypto-map)# exit

! تطبيق خريطة التشفير على الواجهة
Router(config)# interface GigabitEthernet0/0
Router(config-if)# crypto map CMAP
Remote Access VPN

يسمح للمستخدمين الفرديين بالاتصال بشبكة الشركة من مواقع بعيدة.

بروتوكولات شائعة:
  • SSL/TLS VPN: يستخدم بروتوكولات الويب الآمنة، سهل الاستخدام عبر المتصفح.
  • IPsec: يوفر أمانًا قويًا ولكن يتطلب برامج عميل خاصة.
  • L2TP/IPsec: يجمع بين L2TP لإنشاء النفق وIPsec للتشفير.

جودة الخدمة (QoS)

جودة الخدمة (QoS) هي مجموعة من التقنيات لإدارة موارد الشبكة وضمان أداء التطبيقات الحساسة.

مفاهيم أساسية في QoS:

  • التصنيف والتمييز (Classification and Marking): تحديد أنواع حركة المرور وتمييزها.
  • الطوابير (Queuing): ترتيب حزم البيانات حسب الأولوية.
  • تشكيل حركة المرور (Traffic Shaping): التحكم في معدل إرسال البيانات.
  • تجنب الازدحام (Congestion Avoidance): منع امتلاء الطوابير.
  • تحديد النطاق الترددي (Policing): فرض حدود على معدل حركة المرور.

نماذج QoS:

1. نموذج الخدمات المتكاملة (IntServ)

يتطلب حجز موارد مسبقًا لكل تدفق، يستخدم بروتوكول RSVP.

المميزات:
  • ضمانات صارمة لجودة الخدمة
  • مناسب للتطبيقات الحساسة للغاية
العيوب:
  • قابلية محدودة للتوسع
  • يتطلب موارد كبيرة

2. نموذج الخدمات المتمايزة (DiffServ)

يصنف حركة المرور إلى فئات ويعاملها وفقًا لذلك، أكثر قابلية للتوسع.

المميزات:
  • قابلية عالية للتوسع
  • سهولة التنفيذ
  • لا يتطلب حجز موارد لكل تدفق
آلية العمل:
  1. تصنيف الحزم عند حافة الشبكة
  2. تمييز الحزم باستخدام حقل DSCP (Differentiated Services Code Point)
  3. معالجة الحزم في الشبكة بناءً على علامات DSCP

تنفيذ QoS في Cisco IOS:

# تعريف فئات حركة المرور
Router(config)# class-map match-all VOICE
Router(config-cmap)# match dscp ef
Router(config-cmap)# exit

Router(config)# class-map match-all VIDEO
Router(config-cmap)# match dscp af41
Router(config-cmap)# exit

Router(config)# class-map match-all CRITICAL-DATA
Router(config-cmap)# match dscp af21
Router(config-cmap)# exit

# تعريف سياسة QoS
Router(config)# policy-map QOS-POLICY
Router(config-pmap)# class VOICE
Router(config-pmap-c)# priority 512
Router(config-pmap-c)# exit
Router(config-pmap)# class VIDEO
Router(config-pmap-c)# bandwidth 1024
Router(config-pmap-c)# exit
Router(config-pmap)# class CRITICAL-DATA
Router(config-pmap-c)# bandwidth 512
Router(config-pmap-c)# exit
Router(config-pmap)# class class-default
Router(config-pmap-c)# fair-queue
Router(config-pmap-c)# exit

# تطبيق السياسة على الواجهة
Router(config)# interface GigabitEthernet0/0
Router(config-if)# service-policy output QOS-POLICY
